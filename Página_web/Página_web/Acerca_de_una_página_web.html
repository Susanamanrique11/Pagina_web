Las Páginas Web

Una página web, es palabras más, palabras menos, un documento que se encuentra en internet, con texto, colores, tipos de fuentes, multimedia, entre otras, que busca comunicar un mensaje...

Ahora... ¿En qué consiste una página web, qué contiene, te puede servir para algo en la vida? Si quieres descubrir la respuesta a todas estas preguntas, continúa leyendo.

Historia de las páginas web

1990

El informático inglés Tim Berners-Lee, trabajador del CERN, en busca de una manera práctica de compartir facilmente la informacion con los demás investigadores, crea el primer servidor de páginas web, como un sistema abierto y libre

1993

Publicación versión 1.0 de Mosaic, un navegador creado en la Universidad de Illinois por Marc Andreessen,único en permitir, incluir imágenes en las páginas web

1994

Se permitió el acceso de particulares y empresas a Internet. La web se convirtió en el servicio más empleado para ofrecer información, eclipsando a servicios competidores como Gopher o WAIS.

1994

Marc Andreessen fundó el navegador Netscape, publicó la versión 1.0

1994

Berners-Lee fundó el World Wide Web Consortium (W3C) como lugar de encuentro de empresas, universidades y organizaciones sin ánimo de lucro.

1995

Windows incluye en sus programas el navegador Internet Explorer, que comienza a competir contra Netscape

1995 - 2000

Internet Explorer y Netscape competían a un ritmo elevado, publicando actualizaciones cada año, incorporando diferentes funciones y comenzando a manejar css

1995

Brian Eitch creó para Netscape 2.0 el lenguaje de programación Javascript, cuyos programas se podían incluir directamente en las páginas web para ser ejecutados por el navegador. Microsoft creó su propia variante parcialmente incompatible. La normalización de Javascript no la llevó a cabo el W3C, sino la organización ECMA

1995

El W3C creó el XML, unas reglas generales para crear nuevos lenguajes de marcas que fueran compatibles entre sí y que se pudieran tratar con las mismas herramientas, para poder satisfacer la necesidad de ampliacion del HTML, para que incluyera gráficos, fórmulas, etc.; sin embargo el HTML no cumplía las nuevas reglas del XML y el W3C planteó reformular el HTML de acuerdo con ellas (ese nuevo lenguaje se llamaría XHTML).

1997

El ECMA empezó a publicar las normas para unificar y desarrollar el lenguaje, bajo el nombre de ECMAScript.

1998

Netscape creó la organización Mozilla, a la que se donó el código fuente del navegador para que fuera publicado como software libre

2000

El W3C decidió apostar por el XHTML y el grupo de trabajo abandonó el desarrollo del HTML. Se publicó XHTML 1, que equivalía a HTML 4.01, pero cumpliendo XML

2000

Internet Explorer 5.5 introdujo en julio de 2000 la función XMLHttpRequest que permitía conexiones asíncronas del navegador con el servidor, muy útiles en el futuro

2000

La guerra de navegadores terminada con la victoria aplastante de Internet Explorer y la desaparición de Netscape. Microsoft decidió que ya no era necesario seguir innovando y no habría nuevas versiones después de Internet Explorer 6, sin embargo, Internet Explorer 6 no cumplía los aspectos más avanzados de las recomendaciones (especialmente de CSS2) ni admitía documentos XML.

2000 - 2004

Dado que Explorer no seguía las recomendaciones del W3C, el trabajo de esta organización dejó de tener sentido, publicaba solo recomendaciones de HTML y CSS, sacando solo algunos lenguajes de marcas basados en XML (SVG, MathML, etc.), pero explorer no los admitía

2000 - 2004

El grupo Mozilla, desarolla el navegador Mozilla Firefox, que respetaba las recomendaciones del W3C y se empezó a convertir en una alternativa a Internet Explorer.

2001

El W3C creó varios grupos dedicados al desarrollo de la Web Semántica, un conjunto de tecnologías para describir y relacionar la información contenida en una página web para su análisis y procesamiento automático.

2004

Se creó la Fundación Mozilla, que en los primeros años se financiaría sobre todo gracias a Google, y que reconvirtió el navegador Mozilla en el navegador Firefox, que empezó a erosionar el dominio de Internet Explorer.

2004

Se lanza GMAIL

2004

La competencia de Firefox obligó a Microsoft a retomar el desarrollo de Internet Explorer, cumpliendo las recomendaciones del W3C y mejorando el interfaz, pero con la limitación de publicar nuevas versiones coincidiendo con nuevas versiones de Windows.

2004

Se creó el WHATWG, un grupo formado por Mozilla, Apple y Opera al margen del W3C, para retomar el desarrollo del HTML que el W3C había abandonado en favor del XHTML, bajo el nombre de HTML 5.

2005

Se bautizó con el nombre de AJAX al conjunto de tecnologías, basadas en el objeto XMLHttpRequest, que permitían la programación de aplicaciones web

2005

Se lanza Google Maps

2007

El W3C reconsideró su posición y volvió a formar un grupo de trabajo sobre HTML, que trabajaría codo con codo con el WHATWG para publicar la recomendación HTML 5

2008

Se detuvo el trabajo en torno a ECMAScript 4 y se retomó el trabajo en una versión menos ambiciosa que acabaría publicándose como ECMAScript 5 en diciembre de 2009

2009

Google publicó su propio navegador, Google Chrome, que añadió más competencia al mercado, introduciendo un nuevo modelo de desarrollo frenético, con versiones cada dos meses (o menos)

2009

Google publicó su propio navegador, Google Chrome, que añadió más competencia al mercado, introduciendo un nuevo modelo de desarrollo frenético, con versiones cada dos meses (o menos)

2009

Ryan Dahl creó Node.js, un entorno de ejecución de JavaScript basado en V8, el motor de JavaScript de Chrome, que permite ejecutar programas JavaScript como aplicaciones independientes del navegador

2010

Firefox adoptó el modelo de Chrome y pasó a publicar siete o más versiones cada año, con lo que los avances llegan rápidamente a los usuarios

2011

El W3C renunció al desarrollo del XHTML y se concentró en el HTML 5

2011

El WHATWG abandonó por su parte el nombre de HTML 5 y pasó a denominarlo simplemente HTML, abandonando la idea de versiones en favor de una norma "líquida", continuamente modificada y mejorada

2013

Firefox anunció asm.js, un subconjunto de JavaScript diseñado para facilitar la conversión de aplicaciones escritas en C/C++ a JavaScript, para permitir su ejecución en los navegadores

2013

Microsoft cumple después de tanto con Explorer 11, todas las recomendaciones HTML 4 y CSS 2 y admitir lenguajes XML como SVG, pero para mayor compatibilidad y aprovechamiento del HTML 5, crea Microsoft Edge, ligado a las nuevas versiones desde Windows 10

2013

El W3C redujo de forma drástica el desarrollo de la Web semántica, cerrando la mayoría de los grupos de trabajo

2015

El W3C publicó la normativa del HTML 5

2015

Se comienza a publicar el CSS Snapshot, en el que se detallan las normas CSS 3 que se consideran estables; desde entonces CSS Snapshot se publica aproximadamente cada dos años

2015 - 2017

Google Chrome derrotó a Internet Explorer, entre otros motivos debido al uso creciente de los teléfonos móviles (dominados por Android e iOS) y al hecho de que los usuarios de Windows 7 no podían usar Edge

2016

Google Chrome es el navegador dominante, pero se sigue actualizando constantemente y el W3C empezó a cerrar todos los grupos de trabajo relacionados con el XML que se usa en muchos sectores pero no en la web

2016

Google Chrome es el navegador dominante, pero se sigue actualizando constantemente y el W3C empezó a cerrar todos los grupos de trabajo relacionados con el XML que se usa en muchos sectores pero no en la web

2017

Firefox y Chrome incorporan WebAssembly, un formato de código binario basado en JavaScript diseñado para facilitar la conversión a JavaScript de programas escritos en otros lenguajes de programación y así permitir su ejecución en los navegadores

2018

Microsoft anunció que las futuras versiones de Edge estarían basadas en Chromium, el motor de Google Chrome; la tercera guerra de los navegadores terminó definitivamente

2019

El W3C aprobó las primeras recomendaciones del WebAssembly, y anuncia que deja definitivamente el desarrollo del HTML en manos del WHATWG. El dominio de Google sobre la web es prácticamente absoluto; Google presenta una versión de Google Earth en WebAssembly

2020

Firefox aceleró su ritmo de publicación de nuevas versiones (una cada cuatro semanas)

2021

Google anunció que a partir del tercer trimestre de ese año publicaría nuevas versiones de Chrome cada cuatro semanas

2022

La ByteCode Alliance, publicó Wasmtime 1.0, un entorno de ejecución de WebAssembly que permite ejecutar programas WebAssembly como aplicaciones independientes del navegador

2023

Disfrutamos de grandes posibilidades a la hora de navegar, más aún con la implementación de la inteligencia artificial en páginas como chat GPT, en Edge con la creación de imágenes, entre otros
